一、Demo 概覽

本 demo 是 Cortex 通用專案管理平台的單檔 HTML 介面原型,提供完整的 UI 互動體驗供
briefing 階段評估。它涵蓋下列六大核心:

    • OIBG RFQ 8-stage flow:Receive RFQ → Q&A Collect → Q&A Feedback → BOM
         提供 → 並行 Collect → BOM Cost Review → RFQ Cost Review → Submit Final
         Quote。

    • Multi-PM 模型:HOST = 業務(主+助理)+ DPM/BPM/MPM/EPM 4 種 PM 各帶自己
         team。

    • 7 預設 channel 戰情會議室(Slack-like):announcement / general / qa-customer /
         engineering / sourcing / factory / cost-review + DM。

    • RACI 任務矩陣:每個任務有 A (Accountable 背鍋) + R (Responsible 實作),通常不同
         人。

    • Stage Gate 業務確認制:stage 不自動跳,所有 task 完成後業務(或業務助理)點「進入下
         一 stage」。

    • Dependency-based deadline:`QA response+1day`、`EE BOM+3days` 相對
         deadline,不是固定日期。

五、RACI 任務角色矩陣

每個任務 / form 欄位有 2 個角色,通常不同人:
    • A (Accountable)= 「向上負責的人」(背鍋)— 該任務沒做好誰要解釋。

七、⭐ 開案 7 步驟 Wizard (v3 NEW)

業務開案不再「一個個 form 慢慢填」。Wizard 7 步驟引導,大量 AI 預填,業務只要 confirm。
整合 AI #1 / #2 / #32 / #37 — 開案時間從 30 分鐘縮短到 5 分鐘。

⭐ 為什麼是 Wizard?
原本業務開案要打開 form,從上到下填 30+ 欄位:讀 RFQ PDF、查歷史、想 PM、設機密、套
流程、決優先序;新人常漏項、欄位填錯。Wizard 拆 7 步,每步只專注 1 件事,AI 在背後自動補
欄位、推薦選項,業務只要 confirm。

Story 4 · Stage Gate 業務確認制 ★

對應 PPT slide 3 · spec Stage Gate

情境:Stage 6 (BOM Cost Review) 內所有 task 完成,系統發訊息到 #announcement 等業務
確認進入 Stage 7。
Demo 中看到的 Stage Gate 機制:

    • Stage 5 並行 Collect 進行中時,黃色 banner 提示「等業務確認進入下一 stage」。
    • 某 stage 所有 task 完成 → stage.status = READY_FOR_GATE。
    • announcement channel 推訊息「Stage X 全部完成,等業務確認」。
    • 業務(Amy)或業務助理(Joy)點「進入下一 stage」→ stage.status = DONE,下一

         stage = ACTIVE。
    • 進入下一 stage 的 on_enter hook 觸發(自動建任務 / 通知)。

⚖ 為什麼業務 Gate?
PM 對各自任務負責(任務完成 = PM 做完該做的)。整個 stage 由業務確認(stage 1, 6, 7, 8 都
是業務 gate)— 對應 OIBG flow 的「集合會議」概念,業務開會確認可以進下一步。業務不在
線時業務助理可代行。
