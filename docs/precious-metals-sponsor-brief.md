### Slide 4|關鍵議題 #1 — ML 技術路線

**問題**:預測模型用什麼技術棧開發?

| 方案 | 預估 MAPE | 新增工時 | 部署成本 | 推薦度 |
|------|----------|---------|---------|-------|
| A. Python FastAPI 子服務 | 1.0-1.5% | 28 人日 | 新 K8s Pod + CI 分支 | ⭐⭐ |
| **B. Node.js tfjs-node(推薦主力)** | **2.0-3.0%** | **12 人日** | **0** | ⭐⭐⭐⭐ |
| **C. 純 LLM 預測(推薦 baseline)** | **2.5-4.5%** | **3 人日** | **0** | ⭐⭐⭐⭐⭐ |
| D. Python MCP Server | 1.0-1.5% | 25 人日 | 獨立容器 | ⭐⭐⭐(備選) |

**我的推薦**:**C → B → (必要時) D**

- Phase 0:走 C(純 LLM)量 baseline MAPE
- Phase 1:若 C 不夠用則升 B(tfjs-node LSTM)
- Phase 3+:若硬需求 MAPE < 1.5% 且有預算,才評估升 D

**反對理由**(公平呈現):若貴公司有現成 Python ML 團隊,方案 A 邊際成本較低。

**Sponsor 決議欄**:

```
□ 同意推薦路徑(C → B → 必要時 D)
□ 改走 A(全力 Python,時程 +4-6 週)
□ 改走 D(Python MCP,時程 +3 週)
□ 需要 30 分鐘技術討論
```

| 選項 | Phase 0 | Phase 1 (MVP) | Phase 2 (Production) |
