# 問題反饋平台 — 測試劇本

> 測試前確認：server `npm run dev` + client `npm run dev` 已啟動

---

## 前置準備

- **管理員帳號**：ADMIN（登入後確認 Sidebar 有「問題反饋」）
- **一般使用者帳號**：任一非 admin 帳號（若無，先在管理後台建一個）
- **開兩個瀏覽器**（或無痕視窗），分別登入管理員與使用者

---

## 一、基本功能（使用者端）

### 1.1 入口驗證
- [ ] Sidebar「更多功能」展開後可看到「問題反饋」（玫紅色 TicketCheck icon）
- [ ] 點擊進入 `/feedback` 頁面，顯示「我的工單」列表（應為空）
- [ ] 右下角看到藍色浮動按鈕 (FAB)

### 1.2 建立工單 — 完整表單
- [ ] 點「建立工單」進入 `/feedback/new`
- [ ] 填寫主旨：`測試工單-操作問題`
- [ ] 選分類：`系統操作問題`
- [ ] 選優先級：`高`
- [ ] 填描述：`登入後畫面空白，請協助處理`
- [ ] 上傳一個檔案（任意圖片或文件）
- [ ] 按 Ctrl+V 貼上一張截圖（先複製任意圖到剪貼簿）
- [ ] 確認預覽區顯示已上傳的檔案 + 貼上的圖
- [ ] 點「送出」→ 應跳轉到工單詳情頁
- [ ] 確認單號格式為 `FB-YYYYMMDDHHmm`
- [ ] 確認顯示：主旨、描述、分類、優先級、狀態=`待處理`、SLA 時限

### 1.3 建立工單 — FAB 快速開單
- [ ] 回到任意頁面（例如 `/chat`）
- [ ] 點右下角 FAB 按鈕
- [ ] 展開迷你表單，填主旨：`FAB測試單`，選分類，填描述
- [ ] 點「送出」→ 跳轉到新工單詳情頁
- [ ] 確認 source 記錄為 `fab`

### 1.4 工單列表
- [ ] 進入 `/feedback`，確認列出剛才建的 2 張工單
- [ ] 用搜尋框搜「FAB」→ 只顯示 FAB 那張
- [ ] 點篩選，選狀態「待處理」→ 應顯示 2 張
- [ ] 點篩選，選優先級「高」→ 應只顯示第一張

## 二、管理員操作

### 2.1 管理員看到工單
- [ ] 管理員瀏覽器進入 `/feedback`
- [ ] 切換到「全部工單」→ 應看到使用者建的 2 張工單
- [ ] 確認通知 badge（Sidebar 問題反饋旁的紅色數字）> 0

### 2.2 接單
- [ ] 點進第一張工單 `測試工單-操作問題`
- [ ] 點「接單」按鈕
- [ ] 確認狀態變為「處理中」
- [ ] 確認「處理者」顯示管理員名稱
- [ ] 確認 SLA「首次回應」顯示 ✅ 達標

### 2.3 即時對話
- [ ] 管理員在對話框輸入：`已收到，正在查看`
- [ ] 點送出（或 Enter）
- [ ] **切換到使用者瀏覽器**：確認即時收到新訊息（WebSocket 推送，無需刷新）
- [ ] 使用者回覆：`好的，謝謝`
- [ ] **切換到管理員瀏覽器**：確認即時收到回覆
- [ ] 管理員勾選「內部備註」→ 輸入：`可能是 cache 問題`→ 送出
- [ ] 確認備註顯示為琥珀色底 + 鎖頭 icon
- [ ] **切換到使用者瀏覽器**：確認看不到這則內部備註

### 2.4 Typing Indicator
- [ ] 管理員開始打字 → 使用者端看到「xxx 處理中...」提示
- [ ] 管理員停止打字 → 提示消失

### 2.5 附件（對話中）
- [ ] 管理員在對話中附加一個檔案
- [ ] 送出 → 確認右側附件列表更新
- [ ] 點附件可下載

---

## 三、狀態流

### 3.1 等待回覆
- [ ] 管理員點「等待回覆」→ 狀態變為 `等待回覆`
- [ ] **使用者端**回覆一條訊息 → 狀態自動變回 `處理中`

### 3.2 結案
- [ ] 管理員點「結案」→ 彈出結案說明 modal
- [ ] 輸入：`已清除快取，問題解決`→ 確認
- [ ] 狀態變為「已解決」
- [ ] **使用者端**：看到狀態變更 + 出現滿意度評分區

### 3.3 滿意度評分
- [ ] **使用者端**：點星星評 5 分，留言「很快就解決了」
- [ ] 確認右側顯示 5 顆金星 + 留言

### 3.4 重開工單
- [ ] **使用者端**：點「重開」→ 確認提示後確認
- [ ] 狀態變為「已重開」
- [ ] 管理員可以繼續對話處理
- [ ] 再次結案 → 等 72 小時後無法重開（此步驟跳過，邏輯已在後端驗證）

### 3.5 關閉工單
- [ ] 管理員對已解決工單點「關閉」
- [ ] 狀態變為「已結案」
- [ ] 對話框消失，無法再新增訊息

## 四、AI 分析

### 4.1 AI 分析觸發
- [ ] 開啟第二張工單 `FAB測試單`（狀態為待處理）
- [ ] 右側面板找到「AI 分析」→ 點按鈕
- [ ] 確認 SSE streaming 即時顯示 AI 回應
- [ ] 回應完成後顯示模型名稱
- [ ] 如有類似歷史工單，顯示「參考工單」來源
- [ ] 點 👍/👎 按鈕回饋

### 4.2 AI 從對話頁開單
- [ ] 進入 `/chat`，跟 AI 對話一輪
- [ ] 在 AI 回覆下方找到「問題反饋」按鈕 → 點擊
- [ ] 自動跳轉到 `/feedback/new`，描述已預填 AI 回覆摘要
- [ ] 送出工單，確認 source 為 `chat_page`

## 五、通知

### 5.1 Email 通知
- [ ] 建一張新工單 → 檢查管理員 email（`ADMIN_NOTIFY_EMAIL`）是否收到通知信
- [ ] 管理員回覆 → 檢查申請者 email 是否收到回覆通知
- [ ] （如無 SMTP 設定，檢查 server console 有 `[Mail] No SMTP config` 警告即可）

### 5.2 站內通知
- [ ] 管理員 Sidebar「問題反饋」旁的紅色 badge 數字正確
- [ ] badge 數字 > 0 → 進入 `/feedback` 後數字減少（已讀）

### 5.3 Webex Bot（如有設定）
- [ ] 在 Webex 群組中發送：`feedback list` → 應列出待處理工單
- [ ] 發送：`feedback view FB-xxxxxxxx` → 應顯示工單詳情
- [ ] 發送：`feedback reply FB-xxxxxxxx 已在 Webex 回覆` → 回到網頁確認訊息出現
- [ ] 發送：`feedback resolve FB-xxxxxxxx 已處理` → 確認工單結案
- [ ] （如無 Webex Bot，跳過此區塊）

## 六、管理後台

### 6.1 進入 Admin → 問題反饋 Tab
- [ ] 管理員進入 `/admin` → 左側找到「問題反饋」tab → 點擊
- [ ] 確認看到三個區塊：統計儀表板 + 分類管理 + SLA 設定

### 6.2 統計儀表板
- [ ] 確認摘要卡片顯示：總數、SLA 達標率、逾期數、AI 解決數、平均滿意度
- [ ] 確認三個長條圖：狀態分布、優先級分布、分類分布
- [ ] 設定日期篩選 → 確認數據更新
- [ ] 點「匯出 Excel」→ 確認下載 .xlsx 檔案
- [ ] 打開 Excel 確認欄位完整（單號、申請者、部門、分類、狀態…）

### 6.3 分類管理
- [ ] 確認列出 6 個預設分類
- [ ] 點「新增」→ 建一個新分類：名稱 `測試分類`、icon `star`
- [ ] 確認列表更新
- [ ] 點編輯 icon → 修改名稱為 `測試分類-改`→ 儲存
- [ ] 點啟/停用 → 確認狀態切換
- [ ] 點刪除 → 確認刪除（需無工單引用）
- [ ] 到新建工單頁面確認分類下拉選單已更新

### 6.4 SLA 設定
- [ ] 確認 4 個優先級的 SLA 時限表格
- [ ] 修改 `緊急` 的首次回應為 0.5 小時 → 儲存
- [ ] 確認存檔成功
- [ ] 改回 1 小時 → 儲存

## 七、SLA 自動化（觀察 server log）

- [ ] 建一張 urgent 工單但不處理
- [ ] 觀察 server log 每 5 分鐘出現 `[FeedbackSLA]` 日誌
- [ ] SLA 到期前 30 分鐘應觸發 warning 通知
- [ ] SLA 到期後應標記 `sla_breached=1`
- [ ] 已解決工單放置 72 小時 → 自動變 closed（長時間測試，可改 cron 間隔加速驗證）

## 九、邊界測試

### 9.1 權限
- [ ] 使用者嘗試進入其他人的工單 URL → 應顯示 403 或無資料
- [ ] 使用者嘗試接單 → 應顯示「需要管理員權限」
- [ ] 使用者嘗試對已結案工單發訊息 → 應顯示錯誤

### 9.2 空白/特殊字元
- [ ] 建工單不填主旨 → 應提示「主旨為必填」
- [ ] 建工單主旨填 `<script>alert(1)</script>` → 確認不執行（XSS 防護）
- [ ] 描述填超長文字（10000 字）→ 確認可正常建立

### 9.3 並發
- [ ] 兩個瀏覽器同時建工單（同一分鐘內）→ 確認單號不衝突（例如 `FB-xxx` 和 `FB-xxx-2`）
